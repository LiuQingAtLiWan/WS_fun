STL源码剖析勘误

最近抽空，读完了当年只读了一半的书——《STL源码剖析》。

之前读过很多侯先生的译作，获益良多。但侯先生自己写书时，似乎没有译书时那么用心，多处都有表述不清，或与事实不符的地方。记录如下

  • 认为distance函数与output iterator可以兼容
    在原书第99页提到

换句话说，当客端调用distance()并使用output iterator或forward iterator 或者 bidirectional iterator时，统统都会传递调用input iterator的那个__distance()函数。

作者这里是想当然了，事实上，因为output iterator不保证支持++操作，所以distance函数不能以它为参数。当客端调用distance()时，是不能使用output iterator的。

  • 对vector::insert_aux的疑惑

在原书122页到123页中，有代码

此处，作者将insert_aux函数误认为push_back ，以为[position, finish)是原vector的备用空间内容，因而有了疑惑。事实上，它应该是原vector插入点之后的有效内容，拷贝操作是必须的。

此处是明显的笔误，if (ite!=0)应该是if (ite!=ilist.end())才对。

  • 对list的sort算法类型错误归类

    原书142页，有如下代码注释

list无法提供随机访问的功能，它不满足快速quick sort的要求，哪怕在内部实现也不行。其实，list内部使用了一种巧妙的归并排序。

  • 对__adjust_heap函数的误解

在原书的178页，有如下注释

此处又是作者想当然了。在前面找洞的过程中，并没有参考value值，因此，无法保证插入后立即满足次序特性，插入后必须重新调用__push_heap调整位置。若真按侯先生的方式改，那就要出bug了。

反面例子很容易构造，将176页heap内的31替换成35,24替换成33，即可验证。

  • 对红黑树插入情况的讨论不全面

原书210页到211页，作者讨论了红黑树插入新节点后，树结构被破坏的四种情况，分别是

  1. s为黑且X为外侧插入
  2. s为黑且X为内测插入
  3. s为红且X为外侧插入，对P和G单旋转后，并改变X的颜色后，GG为黑
  4. s为红且X为外侧插入，对P和G单旋转后，并改变X的颜色后，GG为红

显然，作者没有分析s为红且X为内侧插入这种情况，如果出现这种情况，可以先按情况2方案处理，处理之后就会转化成情况3或者情况4。

  • 对红黑树的重新平衡调整解析错误

原书212页提到

5.2.2 一个由上而下的程序

并配了图解析。

但实际上，参考225页的源码就知道，所谓“由上而下”，并不是如作者所述的那样，它不是从根部往下处理，它的“由上而下”只体现在待平衡点X和它祖父节点G之间的局部，整体而言，这仍是从下而上的流程。

另外，此处还有明显笔误，“但是如果A的父节点P亦为红色”，这里的“A”应该是X。

  • 对红黑树__insert参数的错误注释
    原书224页，有这样的注释代码

“参数x_为新值插入点”这个说法与事实不符，看后面的源码可以发现，新值插入点是函数内部生成的z = create_node(v);，至于x_，从外部调用看，它只可能是null或者与y_相等。

  • 未理源码解辅助函数的动机，并给了一个不符合c++规范的，更糟糕的替代
    原书300页，有这样的注释

事实上，iterator_traits<InputIterator>::value_type value = *first;这样的写法是不符合c++标准的，标准的写法是typename iterator_traits<InputIterator>::value_type value = *first,如果侯先生真证实这个写法，那他验证时用的应该是一个未跟标准的编译器。

此外，类似iterator_traits<InputIterator>::value_type value = *first这样的代码信息密度低，并不利于阅读，远不如原本代码里value_type(first)这样的表达来得简洁，清晰。

  • 未深究iota名字的含义

原书305页，有这样的注释

iota并不是什么东西的缩写，它就代表第九个希腊字母（ι，中文读约塔），这个函数的命名，源自另一个编程语言——APL。

  • 对nth_element的描述不严谨

原书374页，有这样的注释

(本例是40)描述不对，正确的元素是序列第（5+1）大元素，本例里应该是22。

同类不严谨甚至可算作错误的描述，也出现在原书409页。

  • 对插入排序的描述与事实不符

原书390页，侯先生写道

insertion Sort以双层循环的形式进行。外循环遍历整个序列，每次迭代决定出一个子区间：内层循环遍历子区间，将子区间的每一个“逆转对(inversion)”倒转过来。所谓“逆转对”是值任何两个迭代器 i, j, i <j 而 *i > *j。一旦不存在“逆转对”，序列即排序完毕。

显然，上面描述的是冒泡排序而不是插入排序。在392页对__unguarded_inear_insert函数做注释时，也错误地将“平移”操作注释成“消除逆转对”。

此外，书中还有其他不算错误，但可以写得更好的地方。

比如，algorithm部分的介绍按字典序（而不是按功能归类这种更好的方式）；又比如，部分函数注释只讲了“做什么”而没有讲“为什么”；再比如，对红黑树的理论基本没有讨论，还有一处我觉得挺可惜的——对__rotate_cycle的随机迭代器版本只贴源码毫无解释（本可借此引出美妙的数学同余关系），等等等等。

按惯例，读书后总要写点读后感。但这次写着写着，却变成了勘误表。有些感慨，但无论如何，这书也让我获益不少，起码，它陪我度过了3个愉快的周末。
